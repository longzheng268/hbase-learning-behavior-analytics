一、项目背景
在线学习平台在运行过程中会持续产生学生登录、课程访问、资源浏览、视频学习、练习提交、作业完成等行为数据。这类数据具有数量大、增长快、时间属性明显、查询维度多等特点，适合作为 HBase 的典型应用场景。
本项目要求学生围绕“学习行为数据”完成 HBase 表结构、列族及 RowKey 设计，并使用 HBase Java API 实现数据写入、查询、更新、删除和批量处理。在此基础上，通过 Web 页面展示查询结果，并完成学习活跃度、资源访问量、学习时长等统计分析。
二、作业目标
1. 能够根据业务场景分析学习行为数据的特点，并判断哪些数据适合存储在 HBase 中。
2. 能够设计合理的 HBase 命名空间、表、列族、列限定符和 RowKey。
3. 能够使用 HBase Java API 完成单条与批量数据写入、Get 查询、Scan 范围查询、更新和删除。
4. 能够围绕学生、课程、行为类型和时间范围等维度设计查询功能。
5. 能够完成基础统计分析与可视化，并说明 HBase 在项目中的实际作用。
   三、总体要求
   项目应形成一个可以运行和演示的学习行为数据存储与分析系统。系统可以模拟在线学习平台产生行为数据，也可以通过前端操作自动记录行为数据。学习行为明细必须存储在 HBase 中，学生、课程、资源等规模较小且关系明确的基础数据可以存储在 MySQL 或 SQLite 中。
1. 突出 HBase：项目重点应放在 HBase 数据模型、RowKey、数据读写和范围查询。
2. 真实可运行：提交的代码应能够按照说明完成部署和启动，不接受仅有界面截图或无法复现的项目。
3. 设计有依据：报告中须解释表结构、列族和 RowKey 的设计理由，并说明其支持的查询场景。
4. 成果可验证：应能够通过系统页面和 HBase Shell 查看数据，验证核心功能确实使用 HBase 完成。
   四、系统功能要求
   1 基础数据管理
   系统应具备必要的基础数据，为学习行为记录提供业务上下文。建议包括：
   • 学生信息：学生编号、姓名、专业、班级等。
   • 课程信息：课程编号、课程名称、任课教师等。
   • 学习资源信息：资源编号、资源名称、资源类型、所属课程等。
   • 行为类型字典：登录、课程访问、资源浏览、视频学习、练习提交、作业提交等。
   说明：基础数据可以使用 MySQL 或 SQLite；学习行为明细数据必须使用 HBase。
   2 学习行为数据采集
   系统至少支持以下 5 类学习行为，建议支持 6 类及以上：
1. 登录平台：记录学生登录时间、设备、来源 IP 或客户端类型。
2. 访问课程：记录学生进入某门课程及离开课程的行为。
3. 浏览资源：记录课件、文档、实验指导书等资源的访问。
4. 观看视频：记录视频开始、暂停、结束及观看时长。
5. 提交练习：记录练习提交时间、得分、是否完成等。
6. 提交作业：记录作业提交、修改与完成状态。
   3 HBase 数据存储
1. 创建项目专用命名空间。
2. 创建学习行为核心表，并合理设计一个或多个列族。
3. 设计能够支持主要查询场景的 RowKey，说明字段顺序及原因。
4. 实现单条 Put 写入和批量 Put 写入。
5. 实现 Get 精确查询和 Scan 范围查询。
6. 实现数据更新与删除，理解 HBase 多版本和删除标记的基本机制。
7. 能够通过 HBase Shell 或系统接口验证数据已经写入 HBase。
   4 学习行为查询
1. 查询指定学生的全部学习行为。
2. 查询指定学生在某一时间范围内的学习行为。
3. 查询指定学生在某门课程中的学习行为。
4. 查询某门课程的学习行为记录。
5. 按照行为类型查询记录，例如只查询视频观看记录。
6. 按照时间倒序显示最近发生的学习行为。
7. 至少实现一种组合条件查询，例如“学生 + 课程 + 时间范围”。
   5 统计分析（至少完成 5 项）
   • 学生登录次数统计。
   • 课程访问次数统计。
   • 学习资源浏览次数统计。
   • 视频学习总时长统计。
   • 练习或作业完成次数统计。
   • 学生学习活跃度排行。
   • 课程访问量排行。
   • 每日或每周学习趋势。
   • 学习时段分布。
   • 一定时间内未学习学生名单或简单学习预警。
   统计分析可以通过 Java 程序扫描 HBase 数据后完成。
   6 Web 页面与可视化
1. 学习行为记录列表及分页或分批加载。
2. 按学生、课程、行为类型和时间范围进行查询的页面。
3. 学生个人学习概况页面。
4. 课程访问或资源访问统计页面。
5. 至少 2 种图表，例如折线图、柱状图或饼图。
   页面要求清晰、可操作即可，不以复杂动画和过度美化作为主要评分依据。
   五、提交成果与格式要求
   在学习通提交压缩包，包括3个文件夹：
1. 项目源代码：包括前端、后端、HBase 操作代码、数据库脚本和配置示例。删除编译产物、依赖缓存、日志及无关大文件。
2. 课程设计报告：内容完整，重点突出 HBase 数据建模、RowKey、读写查询、测试结果和问题分析。（使用提供的模板封面）
3. 项目演示视频：建议 5～8 分钟，清晰展示启动过程、数据生成、HBase 写入、条件查询、统计分析与可视化。
