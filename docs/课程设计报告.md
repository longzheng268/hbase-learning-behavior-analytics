# HBase 大数据技术课程设计报告

## 学习行为数据存储与分析系统

## 1. 项目背景与需求分析

### 1.1 项目背景

在线学习平台在运行过程中会持续产生学生登录、课程访问、资源浏览、视频学习、练习提交、作业完成等行为数据。这类数据具有数量大、增长快、时间属性明显、查询维度多等特点，适合作为 HBase 的典型应用场景。

本项目要求围绕"学习行为数据"完成 HBase 表结构、列族及 RowKey 设计，并使用 HBase Java API 实现数据写入、查询、更新、删除和批量处理。在此基础上，通过 Web 页面展示查询结果，并完成学习活跃度、资源访问量、学习时长等统计分析。

### 1.2 需求分析

**功能需求：**

| 需求类别 | 具体需求 |
|---------|---------|
| 基础数据管理 | 学生、课程、学习资源、行为类型字典的管理 |
| 行为数据采集 | 登录、课程访问、资源浏览、视频学习、练习提交、作业提交（6类） |
| HBase 数据存储 | 命名空间、表、列族、RowKey 设计；单条/批量写入；Get/Scan查询；更新删除 |
| 多维查询 | 按学生、课程、行为类型、时间范围及组合条件查询 |
| 统计分析 | 登录次数、课程访问量、视频时长、活跃度排行等≥5项统计 |
| 可视化展示 | Web页面展示查询结果，至少2种图表（折线图、柱状图、饼图） |

**非功能需求：**
- 系统真实可运行，能通过 HBase Shell 验证数据
- 设计有依据，RowKey 设计需说明理由
- 前后端分离，前端页面清晰可操作

## 4. HBase 表结构与列族设计

### 4.1 命名空间

```
命名空间: learning
```

**设计理由：** 使用项目专用命名空间隔离本项目数据与其他HBase数据，便于管理和权限控制。

### 4.2 表结构

```
表名: learning:learning_behavior
列族1: info     — 存储核心索引字段（用于查询和过滤）
列族2: detail   — 存储详细信息（用于展示和分析）
```

### 4.3 列族与列限定符

| 列族 | 列限定符 | 数据类型 | 说明 | 用途 |
|------|---------|---------|------|------|
| info | student_id | String | 学生编号 | 查询索引、RowKey组成部分 |
| info | course_id | String | 课程编号 | 查询索引、Filter过滤 |
| info | behavior_type | String | 行为类型编码 | 查询索引、Filter过滤 |
| info | timestamp | long | 行为发生时间戳 | RowKey组成部分、时间查询 |
| detail | resource_id | String | 学习资源ID | 资源相关行为的附加信息 |
| detail | device | String | 设备信息 | 登录行为的设备记录 |
| detail | ip_address | String | IP地址 | 登录行为的来源记录 |
| detail | score | double | 分数 | 练习/作业提交的得分 |
| detail | duration | long | 时长(秒) | 视频学习的观看时长 |
| detail | status | String | 状态 | 各类行为的状态标记 |
| detail | extra_info | String | 附加JSON | 预留的扩展字段 |

### 4.4 双列族设计理由

1. **info 列族**：存放高频查询的索引字段（student_id, course_id, behavior_type, timestamp）。HBase 在 Scan 时可以只读取 info 列族，减少 I/O 开销。
2. **detail 列族**：存放低频访问的详细字段（设备、IP、分数、时长等）。不同行为类型的 detail 字段差异很大（如视频有时长、练习有分数），使用独立列族可以避免列间的 I/O 干扰。

**分离的好处：**
- 当只需要查询行为列表（不关心详情）时，只加载 info 列族，提高查询效率
- detail 列族的字段按需读取，适合 HBase 的列式存储特性
- 两个列族可以独立配置压缩策略和版本数

## 5. RowKey 设计及热点问题分析

### 5.1 RowKey 格式

```
{studentId}_{reverseTimestamp}_{behaviorType}_{randomSuffix}

示例: S0001_9223372032560558807_LOGIN_0453
      S0003_9223372032555059122_VIDEO_WATCH_8821
```

### 5.2 字段说明

| 字段 | 说明 | 作用 |
|------|------|------|
| studentId | 学生编号，如 S0001 | 作为 RowKey 前缀，支持按学生前缀扫描 |
| reverseTimestamp | Long.MAX_VALUE - timestamp | 将时间倒序排列，最新数据排在前面 |
| behaviorType | 行为类型枚举名 | 区分同一学生的不同行为，支持按类型扫描 |
| randomSuffix | 4位随机数 | 避免同一学生同一毫秒产生重复RowKey |

### 5.3 设计理由

**为什么 studentId 在最前面？**
- 最常用的查询是"查询某学生的全部行为"，studentId 作为前缀可以利用 HBase 的字典序排列，通过 `startRow`/`stopRow` 实现高效前缀扫描
- 避免全表扫描，查询复杂度从 O(N) 降低到 O(M)，M 为该学生的行为数

**为什么使用 reverseTimestamp？**
- HBase RowKey 按字典序排列，直接使用 timestamp 会导致最新数据排在最后
- 使用 `Long.MAX_VALUE - timestamp` 后，最新数据排在前面，符合"按时间倒序显示"的需求
- 支持时间范围查询：通过计算 reverseTimestamp 的范围构造 startRow/stopRow

**为什么添加 randomSuffix？**
- 防止同一学生在同一毫秒内产生多条行为导致 RowKey 重复
- 在一定程度上避免热点写入（同一前缀的数据分散到不同 Region）

### 5.4 热点问题分析

**潜在热点问题：**
- 如果某学生短时间内大量写入（如视频播放期间每秒产生事件），可能导致数据集中在少数 Region
- 学生编号连续分配（S0001-S0020），新注册学生数据可能集中在某个 Region

**缓解措施：**
1. **随机后缀**：4位随机数 (0000-9999) 将同一学生的数据分散到更广的键空间
2. **批量写入**：使用 `table.put(List<Put>)` 批量提交，减少 RPC 次数
3. **盐值前缀（可优化）**：对于更大规模数据，可以对 studentId 做哈希取模，如 `hash(studentId) % N` 作为前缀，但会牺牲前缀查询的效率

**本项目的权衡：**
在"按学生查询"为主要场景的前提下，将 studentId 放在前缀位置是最优选择。randomSuffix 在保证 RowKey 唯一性的同时提供了基本的热点分散能力。

## 6. HBase 数据写入、查询、更新和删除代码实现

### 6.1 单条 Put 写入

```java
public void putBehavior(LearningBehavior b) throws IOException {
    try (Table table = getTable()) {
        Put put = new Put(Bytes.toBytes(b.getRowKey()));
        put.addColumn(cfInfo(), Bytes.toBytes("student_id"), Bytes.toBytes(b.getStudentId()));
        put.addColumn(cfInfo(), Bytes.toBytes("course_id"), Bytes.toBytes(b.getCourseId()));
        put.addColumn(cfInfo(), Bytes.toBytes("behavior_type"), Bytes.toBytes(b.getBehaviorType()));
        put.addColumn(cfInfo(), Bytes.toBytes("timestamp"), Bytes.toBytes(String.valueOf(b.getTimestamp())));
        if (b.getResourceId() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("resource_id"), Bytes.toBytes(b.getResourceId()));
        if (b.getDevice() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("device"), Bytes.toBytes(b.getDevice()));
        if (b.getIpAddress() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("ip_address"), Bytes.toBytes(b.getIpAddress()));
        if (b.getScore() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("score"), Bytes.toBytes(String.valueOf(b.getScore())));
        if (b.getDuration() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("duration"), Bytes.toBytes(String.valueOf(b.getDuration())));
        if (b.getStatus() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("status"), Bytes.toBytes(b.getStatus()));
        table.put(put);
    }
}
```

**说明：** 每条 Put 操作包含 RowKey 和两个列族的数据。条件写入（`if != null`）避免写入空列，节省存储空间。

### 6.2 批量 Put 写入

```java
public void putBehaviors(List<LearningBehavior> behaviors) throws IOException {
    try (Table table = getTable()) {
        List<Put> puts = new ArrayList<>();
        for (LearningBehavior b : behaviors) {
            puts.add(buildPut(b));
        }
        table.put(puts); // 批量提交
    }
}
```

**说明：** 使用 `table.put(List<Put>)` 进行批量写入，相比逐条 Put 减少了 RPC 往返次数，显著提升写入吞吐量。在数据生成器中，每累积 500 条记录执行一次批量写入。

### 6.3 Get 精确查询

```java
public LearningBehavior getByRowKey(String rowKey) throws IOException {
    try (Table table = getTable()) {
        Get get = new Get(Bytes.toBytes(rowKey));
        Result result = table.get(get);
        if (result.isEmpty()) return null;
        return resultToBehavior(result);
    }
}
```

**说明：** Get 操作通过精确的 RowKey 定位唯一记录，时间复杂度 O(1)。同时加载两个列族的数据以获取完整信息。

### 6.4 Scan 范围查询

```java
// 按学生查询
public List<LearningBehavior> scanByStudent(String studentId) throws IOException {
    try (Table table = getTable()) {
        Scan scan = new Scan();
        scan.withStartRow(scanStartRow(studentId));   // studentId + "_"
        scan.withStopRow(scanStopRow(studentId));     // studentId + "`"
        return doScan(table, scan);
    }
}

// 按学生+课程查询（使用 Filter）
public List<LearningBehavior> scanByStudentAndCourse(
        String studentId, String courseId) throws IOException {
    try (Table table = getTable()) {
        Scan scan = new Scan();
        scan.withStartRow(scanStartRow(studentId));
        scan.withStopRow(scanStopRow(studentId));
        SingleColumnValueFilter courseFilter = new SingleColumnValueFilter(
                cfInfo(), Bytes.toBytes("course_id"),
                CompareOperator.EQUAL, Bytes.toBytes(courseId));
        courseFilter.setFilterIfMissing(true);
        scan.setFilter(courseFilter);
        return doScan(table, scan);
    }
}
```

**说明：**
- `withStartRow`/`withStopRow` 利用 RowKey 前缀实现范围扫描
- `SingleColumnValueFilter` 在服务端过滤不需要的数据，减少网络传输
- `setFilterIfMissing(true)` 确保只有包含该列的行才会被返回

### 6.5 数据更新

```java
public void updateBehavior(String rowKey, Map<String, String> updates) throws IOException {
    try (Table table = getTable()) {
        Get get = new Get(Bytes.toBytes(rowKey));
        Result result = table.get(get);
        if (result.isEmpty()) throw new IOException("Row not found: " + rowKey);

        Put put = new Put(Bytes.toBytes(rowKey));
        for (Map.Entry<String, String> entry : updates.entrySet()) {
            byte[] qualifier = Bytes.toBytes(entry.getKey());
            byte[] value = Bytes.toBytes(entry.getValue());
            if (isInfoColumn(entry.getKey())) {
                put.addColumn(cfInfo(), qualifier, value);
            } else {
                put.addColumn(cfDetail(), qualifier, value);
            }
        }
        table.put(put); // HBase 的 Put 本身就是 upsert 操作
    }
}
```

**说明：** HBase 的更新机制是"追加写入"（LSM-Tree），新的 Put 会覆盖同一 RowKey+列族+列限定符 的旧值。旧版本数据在 Major Compaction 时才会物理删除。

### 6.6 数据删除

```java
public void deleteBehavior(String rowKey) throws IOException {
    try (Table table = getTable()) {
        table.delete(new Delete(Bytes.toBytes(rowKey)));
    }
}
```

**说明：** HBase 的删除操作实际上是写入一个"删除标记"（Delete Marker/Tombstone）。在后续读取时，带有删除标记的版本会被过滤。只有在 Major Compaction 后，被标记的数据才会被物理清除。这是 HBase 的多版本机制和 LSM-Tree 架构决定的。

### 6.7.2 HBase 连接配置 HBaseConfig.java

```java
package com.learning.config;

import org.apache.hadoop.hbase.HBaseConfiguration;
import org.apache.hadoop.hbase.client.Connection;
import org.apache.hadoop.hbase.client.ConnectionFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import javax.annotation.PreDestroy;
import java.io.IOException;

@Configuration
public class HBaseConfig {

    @Value("${hbase.zookeeper.quorum:localhost}")
    private String zookeeperQuorum;

    @Value("${hbase.zookeeper.port:2181}")
    private int zookeeperPort;

    private Connection connection;

    @Bean
    public org.apache.hadoop.conf.Configuration hbaseConfiguration() {
        org.apache.hadoop.conf.Configuration config = HBaseConfiguration.create();
        config.set("hbase.zookeeper.quorum", zookeeperQuorum);
        config.setInt("hbase.zookeeper.property.clientPort", zookeeperPort);
        return config;
    }

    @Bean
    public Connection hbaseConnection() throws IOException {
        if (connection == null || connection.isClosed()) {
            connection = ConnectionFactory.createConnection(hbaseConfiguration());
        }
        return connection;
    }

    @PreDestroy
    public void destroy() throws IOException {
        if (connection != null && !connection.isClosed()) {
            connection.close();
        }
    }
}
```

### 6.7.3 行为类型枚举 BehaviorType.java

```java
package com.learning.model;

public enum BehaviorType {
    LOGIN("登录"),
    COURSE_ACCESS("课程访问"),
    RESOURCE_BROWSE("资源浏览"),
    VIDEO_WATCH("视频学习"),
    QUIZ_SUBMIT("练习提交"),
    HOMEWORK_SUBMIT("作业提交");

    private final String displayName;

    BehaviorType(String displayName) {
        this.displayName = displayName;
    }

    public String getDisplayName() {
        return displayName;
    }

    public static BehaviorType fromString(String text) {
        for (BehaviorType b : BehaviorType.values()) {
            if (b.name().equalsIgnoreCase(text) || b.displayName.equals(text)) {
                return b;
            }
        }
        return null;
    }
}
```

```java
package com.learning.dao;

import com.learning.model.BehaviorType;
import com.learning.model.LearningBehavior;
import org.apache.hadoop.hbase.CompareOperator;
import org.apache.hadoop.hbase.TableName;
import org.apache.hadoop.hbase.client.*;
import org.apache.hadoop.hbase.filter.SingleColumnValueFilter;
import org.apache.hadoop.hbase.util.Bytes;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Repository;

import javax.annotation.PostConstruct;
import java.io.IOException;
import java.util.*;

@Repository
public class HBaseDao {

    @Autowired
    private Connection connection;

    @Value("${hbase.namespace:learning}")
    private String namespace;

    private static final String TABLE_NAME_STR = "learning_behavior";
    private static final String CF_INFO_STR = "info";
    private static final String CF_DETAIL_STR = "detail";

    private byte[] cfInfo() { return Bytes.toBytes(CF_INFO_STR); }
    private byte[] cfDetail() { return Bytes.toBytes(CF_DETAIL_STR); }

    public String getFullTableName() {
        return namespace + ":" + TABLE_NAME_STR;
    }

    public Table getTable() throws IOException {
        return connection.getTable(TableName.valueOf(namespace, TABLE_NAME_STR));
    }

    @PostConstruct
    public void initTable() throws IOException {
        Admin admin = connection.getAdmin();
        TableName tn = TableName.valueOf(namespace, TABLE_NAME_STR);
        try {
            admin.getNamespaceDescriptor(namespace);
        } catch (Exception e) {
            admin.createNamespace(
                org.apache.hadoop.hbase.NamespaceDescriptor.create(namespace).build());
        }
        if (!admin.tableExists(tn)) {
            TableDescriptorBuilder builder = TableDescriptorBuilder.newBuilder(tn);
            builder.setColumnFamily(ColumnFamilyDescriptorBuilder.of(cfInfo()));
            builder.setColumnFamily(ColumnFamilyDescriptorBuilder.of(cfDetail()));
            admin.createTable(builder.build());
        }
        admin.close();
    }

    // ======================== RowKey ========================

    public String generateRowKey(String studentId, long timestamp, BehaviorType behaviorType) {
        long reverseTs = Long.MAX_VALUE - timestamp;
        String randomSuffix = String.format("%04d", new Random().nextInt(10000));
        return String.format("%s_%019d_%s_%s", studentId, reverseTs, behaviorType.name(), randomSuffix);
    }

    private byte[] scanStartRow(String studentId) {
        return Bytes.toBytes(studentId + "_");
    }

    private byte[] scanStopRow(String studentId) {
        return Bytes.toBytes(studentId + "`");
    }

    private byte[] timeStartRow(String studentId, long endTimestamp) {
        long reverseEnd = Long.MAX_VALUE - endTimestamp;
        return Bytes.toBytes(String.format("%s_%019d_", studentId, reverseEnd));
    }

    private byte[] timeStopRow(String studentId, long startTimestamp) {
        long reverseStart = Long.MAX_VALUE - startTimestamp;
        return Bytes.toBytes(String.format("%s_%019d`", studentId, reverseStart));
    }

    // ======================== 单条写入 ========================

    public void putBehavior(LearningBehavior b) throws IOException {
        try (Table table = getTable()) {
            table.put(buildPut(b));
        }
    }

    // ======================== 批量写入 ========================

    public void putBehaviors(List<LearningBehavior> behaviors) throws IOException {
        try (Table table = getTable()) {
            List<Put> puts = new ArrayList<>();
            for (LearningBehavior b : behaviors) {
                puts.add(buildPut(b));
            }
            table.put(puts);
        }
    }

    private Put buildPut(LearningBehavior b) {
        Put put = new Put(Bytes.toBytes(b.getRowKey()));
        put.addColumn(cfInfo(), Bytes.toBytes("student_id"), Bytes.toBytes(b.getStudentId()));
        put.addColumn(cfInfo(), Bytes.toBytes("course_id"), Bytes.toBytes(b.getCourseId()));
        put.addColumn(cfInfo(), Bytes.toBytes("behavior_type"), Bytes.toBytes(b.getBehaviorType()));
        put.addColumn(cfInfo(), Bytes.toBytes("timestamp"), Bytes.toBytes(String.valueOf(b.getTimestamp())));
        if (b.getResourceId() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("resource_id"), Bytes.toBytes(b.getResourceId()));
        if (b.getDevice() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("device"), Bytes.toBytes(b.getDevice()));
        if (b.getIpAddress() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("ip_address"), Bytes.toBytes(b.getIpAddress()));
        if (b.getScore() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("score"), Bytes.toBytes(String.valueOf(b.getScore())));
        if (b.getDuration() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("duration"), Bytes.toBytes(String.valueOf(b.getDuration())));
        if (b.getStatus() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("status"), Bytes.toBytes(b.getStatus()));
        if (b.getExtraInfo() != null)
            put.addColumn(cfDetail(), Bytes.toBytes("extra_info"), Bytes.toBytes(b.getExtraInfo()));
        return put;
    }

    // ======================== Get ========================

    public LearningBehavior getByRowKey(String rowKey) throws IOException {
        try (Table table = getTable()) {

            Result result = table.get(get);

        try (Table table = getTable()) {
            Scan scan = new Scan();
            scan.withStartRow(timeStartRow(studentId, endTs));
            scan.withStopRow(timeStopRow(studentId, startTs));
            SingleColumnValueFilter filter = new SingleColumnValueFilter(
                    cfInfo(), Bytes.toBytes("course_id"),

    // ======================== 更新 ========================

    public void updateBehavior(String rowKey, Map<String, String> updates) throws IOException {
        try (Table table = getTable()) {
            Get get = new Get(Bytes.toBytes(rowKey));
            Result result = table.get(get);
            if (result.isEmpty()) throw new IOException("Row not found: " + rowKey);

            Put put = new Put(Bytes.toBytes(rowKey));
            for (Map.Entry<String, String> entry : updates.entrySet()) {
                byte[] qualifier = Bytes.toBytes(entry.getKey());
                byte[] value = Bytes.toBytes(entry.getValue());
                if (isInfoColumn(entry.getKey())) {
                    put.addColumn(cfInfo(), qualifier, value);
                } else {
                    put.addColumn(cfDetail(), qualifier, value);
                }
            }
            table.put(put);
        }
    }

    private boolean isInfoColumn(String col) {
        return "student_id".equals(col) || "course_id".equals(col) ||
               "behavior_type".equals(col) || "timestamp".equals(col);
    }

    // ======================== 删除 ========================

    public void deleteBehavior(String rowKey) throws IOException {
        try (Table table = getTable()) {
            table.delete(new Delete(Bytes.toBytes(rowKey)));
        }
    }

    // ======================== 辅助 ========================

    private List<LearningBehavior> doScan(Table table, Scan scan) throws IOException {
        List<LearningBehavior> list = new ArrayList<>();
        try (ResultScanner scanner = table.getScanner(scan)) {
            for (Result result : scanner) {
                LearningBehavior b = resultToBehavior(result);
                if (b != null) list.add(b);
            }
        }
        return list;
    }

    private LearningBehavior resultToBehavior(Result r) {
        if (r.isEmpty()) return null;
        LearningBehavior b = new LearningBehavior();
        b.setRowKey(Bytes.toString(r.getRow()));
        b.setStudentId(bs(r, cfInfo(), Bytes.toBytes("student_id")));
        b.setCourseId(bs(r, cfInfo(), Bytes.toBytes("course_id")));
        b.setBehaviorType(bs(r, cfInfo(), Bytes.toBytes("behavior_type")));
        String tsStr = bs(r, cfInfo(), Bytes.toBytes("timestamp"));
        if (tsStr != null) b.setTimestamp(Long.parseLong(tsStr));
        b.setResourceId(bs(r, cfDetail(), Bytes.toBytes("resource_id")));
        b.setDevice(bs(r, cfDetail(), Bytes.toBytes("device")));
        b.setIpAddress(bs(r, cfDetail(), Bytes.toBytes("ip_address")));
        b.setStatus(bs(r, cfDetail(), Bytes.toBytes("status")));
        b.setExtraInfo(bs(r, cfDetail(), Bytes.toBytes("extra_info")));
        String scoreStr = bs(r, cfDetail(), Bytes.toBytes("score"));
        if (scoreStr != null) b.setScore(Double.parseDouble(scoreStr));
        String durStr = bs(r, cfDetail(), Bytes.toBytes("duration"));
        if (durStr != null) b.setDuration(Long.parseLong(durStr));
        return b;
    }

    private String bs(Result r, byte[] cf, byte[] q) {
        byte[] val = r.getValue(cf, q);
        return val != null ? Bytes.toString(val) : null;
    }
}
```

```java
package com.learning.service;

import com.learning.dao.HBaseDao;
import com.learning.model.LearningBehavior;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.io.IOException;
import java.text.SimpleDateFormat;
import java.util.*;
import java.util.stream.Collectors;

@Service
public class StatisticsService {

    @Autowired
    private HBaseDao hbaseDao;

    public Map<String, Long> studentLoginCount() throws IOException {
        return hbaseDao.scanAll().stream()
                .filter(b -> "LOGIN".equals(b.getBehaviorType()))
                .collect(Collectors.groupingBy(LearningBehavior::getStudentId, Collectors.counting()));
    }

    public Map<String, Long> courseAccessCount() throws IOException {
        return hbaseDao.scanAll().stream()
                .filter(b -> "COURSE_ACCESS".equals(b.getBehaviorType()))
                .collect(Collectors.groupingBy(LearningBehavior::getCourseId, Collectors.counting()));
    }

    public Map<String, Long> resourceBrowseCount() throws IOException {
        return hbaseDao.scanAll().stream()
                .filter(b -> "RESOURCE_BROWSE".equals(b.getBehaviorType())
                        && b.getResourceId() != null)
                .collect(Collectors.groupingBy(LearningBehavior::getResourceId, Collectors.counting()));
    }

    public Map<String, Long> videoWatchDurationByCourse() throws IOException {
        return hbaseDao.scanAll().stream()
                .filter(b -> "VIDEO_WATCH".equals(b.getBehaviorType()) && b.getDuration() != null)
                .collect(Collectors.groupingBy(LearningBehavior::getCourseId,
                        Collectors.summingLong(LearningBehavior::getDuration)));
    }

    public Map<String, Long> submitCountByType() throws IOException {
        return hbaseDao.scanAll().stream()
                .filter(b -> "QUIZ_SUBMIT".equals(b.getBehaviorType())
                        || "HOMEWORK_SUBMIT".equals(b.getBehaviorType()))
                .collect(Collectors.groupingBy(LearningBehavior::getBehaviorType, Collectors.counting()));
    }

    public List<Map<String, Object>> studentActivityRanking() throws IOException {
        Map<String, Long> counts = hbaseDao.scanAll().stream()
                .collect(Collectors.groupingBy(LearningBehavior::getStudentId, Collectors.counting()));
        List<Map<String, Object>> ranking = new ArrayList<>();
        counts.entrySet().stream()
                .sorted(Map.Entry.<String, Long>comparingByValue().reversed())
                .forEach(e -> {
                    Map<String, Object> item = new LinkedHashMap<>();
                    item.put("studentId", e.getKey());
                    item.put("count", e.getValue());
                    ranking.add(item);
                });
        return ranking;
    }

    public Map<String, Long> dailyTrend() throws IOException {
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd");
        return hbaseDao.scanAll().stream()
                .collect(Collectors.groupingBy(
                        b -> sdf.format(new Date(b.getTimestamp())),
                        TreeMap::new, Collectors.counting()));
    }

    public Map<Integer, Long> hourlyDistribution() throws IOException {
        return hbaseDao.scanAll().stream()
                .collect(Collectors.groupingBy(
                        b -> {
                            Calendar cal = Calendar.getInstance();
                            cal.setTimeInMillis(b.getTimestamp());
                            return cal.get(Calendar.HOUR_OF_DAY);
                        },
                        TreeMap::new, Collectors.counting()));
    }

    public List<String> inactiveStudents(List<String> allStudentIds, int days) throws IOException {
        long cutoff = System.currentTimeMillis() - (long) days * 24 * 3600 * 1000;
        Set<String> activeStudents = hbaseDao.scanAll().stream()
                .filter(b -> b.getTimestamp() >= cutoff)
                .map(LearningBehavior::getStudentId)
                .collect(Collectors.toSet());
        return allStudentIds.stream()
                .filter(id -> !activeStudents.contains(id))
                .collect(Collectors.toList());
    }
```

```java
                Map<String, Object> item = new LinkedHashMap<>();
                item.put("studentId", e.getKey());
                item.put("count", e.getValue());
                ranking.add(item);
            });
    return ranking;
}

// 学习预警：查找N天内未活跃的学生
public List<String> inactiveStudents(List<String> allStudentIds, int days) throws IOException {
    long cutoff = System.currentTimeMillis() - (long) days * 24 * 3600 * 1000;
    Set<String> activeStudents = hbaseDao.scanAll().stream()
            .filter(b -> b.getTimestamp() >= cutoff)
            .map(LearningBehavior::getStudentId)
            .collect(Collectors.toSet());
    return allStudentIds.stream()
            .filter(id -> !activeStudents.contains(id))
            .collect(Collectors.toList());
}
```

## 9. 项目总结、不足与改进方向

### 9.1 项目总结

本项目围绕"学习行为数据"完成了基于 HBase 的数据存储与分析系统，主要成果包括：

1. **合理的 HBase 数据模型设计**：使用 learning 命名空间、learning_behavior 表、info/detail 双列族，将索引字段与详细信息分离存储
2. **精心设计的 RowKey**：采用 `{studentId}_{reverseTimestamp}_{behaviorType}_{randomSuffix}` 格式，高效支持按学生查询、时间范围查询、时间倒序排列
3. **完整的 HBase CRUD 操作**：实现了单条/批量 Put、Get 精确查询、Scan 范围查询（含 Filter）、数据更新和删除
4. **10 项统计分析**：涵盖登录次数、课程访问、资源浏览、视频时长、提交统计、活跃度排行、每日趋势、时段分布、学习预警等
5. **完整的 Web 界面**：Java Swing 风格设计，6 个功能页面，10+ 种 ECharts 图表
6. **真实可运行**：系统可通过浏览器操作，数据可通过 HBase Shell 验证

### 9.2 不足之处

1. **统计分析效率**：当前采用全表扫描+Java聚合的方式，数据量增大后性能会下降
2. **缺少认证鉴权**：Web 系统无用户登录功能，存在数据安全风险
3. **实时性不足**：统计结果是实时计算的，没有缓存机制，频繁查询会增加 HBase 压力
4. **RowKey 设计局限**：按课程查询需要全表扫描+Filter，无法利用 RowKey 有序性
5. **缺少数据导入导出**：不支持从 CSV/JSON 批量导入历史数据

### 9.3 改进方向

1. **引入二级索引**：使用 HBase Coprocessor 或 Phoenix 实现按 courseId 的索引查询
2. **使用 MapReduce/Spark**：对大规模数据的统计分析使用分布式计算框架
3. **增加缓存层**：引入 Redis 缓存热点查询结果和统计数据
4. **多 RowKey 设计**：创建辅助表，以 `{courseId}_{timestamp}` 为 RowKey 支持按课程高效查询
5. **引入 Kafka**：实现实时数据流采集，支持更及时的行为分析
6. **使用 Phoenix**：提供 SQL 接口访问 HBase，降低开发复杂度

### C. HBase 关键概念说明

| 概念 | 说明 |
|------|------|
| 命名空间 (Namespace) | 类似于关系数据库的 schema，用于逻辑隔离表 |
| 列族 (Column Family) | 列的逻辑分组，存储在同一 HFile 中 |
| 列限定符 (Column Qualifier) | 列族下的具体列名，可以动态添加 |
| RowKey | 行键，表中唯一标识一行，按字典序排列 |
| 多版本 (Multi-Version) | HBase 默认保留3个版本的数据，可以配置 |
| 删除标记 (Delete Marker) | 删除操作写入的标记，Major Compaction 时物理删除 |
| Region | 表的水平分区，是负载均衡和数据分片的基本单位 |
| LSM-Tree | HBase 的存储结构，写入先进 MemStore，后刷写到 HFile |
